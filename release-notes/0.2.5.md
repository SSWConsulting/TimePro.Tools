# 0.2.5

- `tp login --api-url` now normalizes local TimePro URLs to the localhost dev
  endpoint, avoiding the local certificate/port mismatch.
- `tp leave create` now supports `--timezone` and uses it before your TimePro
  profile timezone, with the machine timezone as the fallback.
- MCP leave creation now follows the same timezone order, so agents can create
  EasyLeave requests without relying on browser timezone state.
